# ADLC Prompt: Requirements Generation for SecureClaims AI

## Project Overview
SecureClaims AI is an AI-powered insurance claims processing platform that automates fraud detection, claim validation, risk assessment, and decision-making using machine learning and natural language processing.

## Prompt Instructions
Using the AI-Driven Development Lifecycle (ADLC), generate detailed, structured requirements for SecureClaims AI covering:

1. **Functional Requirements** — What the system must do
2. **Non-Functional Requirements** — Performance, security, scalability, compliance
3. **User Roles & Permissions** — Who interacts with the system
4. **System Integrations** — External systems and APIs
5. **AI/ML Requirements** — Model capabilities, training, and inference
6. **Data Requirements** — Data ingestion, storage, and governance
7. **Compliance & Regulatory** — HIPAA, GDPR, SOC2, insurance regulations
8. **Constraints & Assumptions** — Known limitations and assumptions

## Output Format
Generate the output as a detailed requirements document saved to `.kiro/outputs/01-requirements-output.md`
